
## Урок 1 — Первое подключение к VPS и настройка SSH

### Что сделал
- Подключился к VPS по SSH под root
- Создал нового пользователя azimjon
- Дал права sudo через usermod -aG
- Настроил SSH-ключи (ed25519)
- Зашёл на сервер без пароля

### Команды которые запомнить
| Команда | Что делает |
|---|---|
| `ssh user@ip` | Подключиться к серверу |
| `adduser имя` | Создать пользователя |
| `usermod -aG sudo имя` | Добавить в группу sudo |
| `id имя` | Проверить группы пользователя |
| `ssh-keygen -t ed25519` | Создать SSH-ключ |
| `ssh-copy-id user@ip` | Скопировать ключ на сервер |

### Важные понятия
- **root** — суперпользователь, работать под ним опасно
- **sudo** — временное повышение прав
- **SSH-ключ** — приватный у меня, публичный на сервере
- **-aG флаг** — добавить к группам, не заменить их

  ## Урок 2 — Обновление системы и защита сервера

### Что сделал
- Обновил систему через apt update && apt upgrade
- Перезагрузил сервер после обновления ядра
- Закрыл вход по паролю (PasswordAuthentication no)
- Закрыл вход под root (PermitRootLogin no)
- Перезапустил SSH и проверил что доступ работает

### Команды
| Команда | Что делает |
|---|---|
| `sudo apt update && sudo apt upgrade -y` | Обновить систему |
| `sudo reboot` | Перезагрузить сервер |
| `uname -r` | Показать версию ядра |
| `sudo nano /etc/ssh/sshd_config` | Открыть конфиг SSH |
| `sudo systemctl restart ssh` | Перезапустить SSH |

### Важные понятия
- **PasswordAuthentication no** — запрещает вход по паролю
- **PermitRootLogin no** — запрещает вход под root
- **sshd_config** — главный конфиг SSH сервера
- Всегда проверяй новый терминал до закрытия старого

### Правило
Никогда не закрывай текущую SSH сессию пока не
проверил что новая сессия подключается успешно
